# Pharmacy Management System - MVP Definition

## 1. Introduction
The MVP (Minimum Viable Product) of the Pharmacy Management System (PMS) defines the **essential features and modules** required for the first version. The goal is to deliver a functional system that solves core business problems while allowing incremental improvements in future releases.

**Objectives:**
- Enable core pharmacy operations
- Ensure inventory and sales tracking
- Support prescription management
- Provide basic reporting
- Allow secure user access

---

## 2. MVP Scope
The MVP will include **only essential modules** necessary for daily pharmacy operations:

### 2.1 User Management
- Add/Edit/Delete users
- Role-based access: Admin, Pharmacist
- Login and logout functionality

### 2.2 Inventory Management
- Add medicines with batch, expiry, and quantity
- Track stock in/out
- Low stock alerts
- Expiry notifications

### 2.3 POS & Sales
- Add items to cart and process payment
- Link prescriptions to sales
- Apply discounts
- Print invoice

### 2.4 Prescription Management
- Add and store prescriptions
- Link prescriptions to sales
- Support basic patient/doctor details

### 2.5 Reporting (Basic)
- Daily sales report
- Stock report (current, low, expired)
- Simple user activity report

### 2.6 Alerts & Notifications
- Low stock notifications
- Expired medicine alerts

### 2.7 Backup & Security
- Role-based access control
- Daily automated backup
- Basic encryption for sensitive data

---

## 3. Excluded from MVP (Future Releases)
- Advanced analytics dashboard
- Multi-branch support
- Supplier & purchase management (full workflow)
- Email/SMS notifications (optional, for now)
- Advanced reporting with charts and graphs
- Offline POS mode
- Cloud integration and mobile app support

---

## 4. MVP Success Criteria
- Pharmacist can process sales and manage inventory efficiently
- Admin can view basic reports and manage users
- Prescriptions can be recorded and linked to sales
- Low stock and expired medicines are tracked and alerted
- System is secure and data is backed up daily

---

## 5. MVP Deliverables
| Module | Deliverable |
|--------|------------|
| User Management | Add/Edit/Delete users, login/logout |
| Inventory Management | Add/update medicines, stock alerts, expiry tracking |
| POS & Sales | Process sales, apply discounts, invoice, prescription link |
| Prescription Management | Add and view prescriptions |
| Reporting | Basic daily sales and stock report |
| Alerts & Notifications | Low stock and expired medicine alerts |
| Backup & Security | Daily backup, role-based access, basic encryption |

---

## 6. Assumptions
- Staff can use basic computer systems
- Pharmacy has at least one PC or tablet for POS
- MVP will be used in a single branch
- Data integrity and backup handled by IT admin

---

## 7. Glossary
- **MVP:** Minimum Viable Product  
- **POS:** Point of Sale  
- **Stock Out:** When a medicine is unavailable  
- **Prescription:** Doctor’s order for medicine  
- **Alerts:** Notifications for low stock or expired medicines
